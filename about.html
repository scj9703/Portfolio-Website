Hi, I'm Sam.

I'm studying Computer Science at the Rochester Institute of Technology, where my love for creating and tinkering has led me to pursue a career in software development. I love learning new things, and I've gathered knowledge on a wide variety of technologies during my studies.


Much of my free time is spent on my game development hobby, where I volunteer as a developer for the JinGames modded Minecraft server. Under my pseudonym 'Mighty', I create multiplayer content that supplements the JinGames Minecraft mods for players to enjoy, largely through the CustomNPCs+ mod for content creation. I also work with support and bug report tickets.


I've used JavaScript ES5 quite a bit in this role, mainly for NPC scripting to create enjoyable boss battles or minigames that go far beyond anything normally possible in plain Minecraft. Other times my scripts are used for server utilities, anti-exploit measures, or data collection.


During my time here I also co-created a mod of my own, 'Spirit Control.' It's an addon for the JinGames core mods, created with Java and built with Gradle. It adds some new features to our combat system, like Super Attacks, Passive Abilities, a Super Attack Meter, and a customizable loadout. It also has JSON config support for easy server-sided balancing, and a scripting API to allow content creators to work with the mod in JavaScript. You can find the mod as well as the aforementioned scripts on my GitHub page.


But that's not all I have experience with. Front-End Development projects are among my favorites, and I've been working to hone my skills in HTML, CSS, and JavaScript. This website in particular was a lot of fun to create. I've also utilized TypeScript and the Angular framework to build the UI for a medical needs application as a term project.


On the same aforementioned project, I worked with our Back-End team to make our application have access to the services provided by our Java Spring REST API. While this was an academic project, we used real industry practices, Agile methodologies, unit testing, and thorough documentation. You can view our design doc here, or check out an overview of the term project here.
